# Политика обновления зависимостей

## Варианты

### Ручное обновление

- Периодически (например раз в квартал) проверять устаревшие пакеты:
  - **npm:** `npm outdated`, обновление по одному или `npm update` для минорных/патчей.
  - **Composer:** `composer outdated`, обновление через `composer require vendor/package:^newversion`.
- После обновления — полный прогон тестов и сборки: `npm run test`, `npm run build`, проверка ключевых страниц.

### Dependabot (GitHub)

- Включить в репозитории: **Settings → Security → Dependabot → Enable**.
- Создать `.github/dependabot.yml` с расписанием (например еженедельно) для `npm` и `composer`.
- PR от Dependabot проверять тестами и сборкой; при необходимости обновлять конфиги (breaking changes).

### Renovate

- Установить приложение Renovate в репозитории.
- Настроить через `renovate.json` или конфиг в репозитории (группировка обновлений, авто-мерж патчей и т.д.).

## Рекомендации

- **Патчи (patch):** применять регулярно, риск минимален.
- **Минорные (minor):** после прогона тестов и краткой проверки.
- **Мажорные (major):** проверять changelog на breaking changes, тесты и ручная проверка критичных сценариев.
- Версии в `composer.json` и `package.json` по возможности указывать с `^` (минор/патч) или `~` (только патч), чтобы получать совместимые обновления.
