# Политика резервного копирования

Рекомендуемые объекты для бэкапов и периодичность.

## Что копировать

| Объект         | Путь / описание                                                                       | Приоритет    |
| -------------- | ------------------------------------------------------------------------------------- | ------------ |
| Контент (JSON) | `data/json/` — страницы, SEO, глобальные настройки, коллекции (например рестораны)    | Высокий      |
| Медиа          | Файлы в `data/img/`, загружаемые пользователями (если есть каталог uploads)           | Высокий      |
| Конфигурация   | `.env` (секреты — хранить отдельно и шифровать), `config/*.php` при кастомных правках | Средний      |
| Логи           | `logs/` — при необходимости разбора инцидентов                                        | Низкий       |
| Кэш            | `cache/` — можно не бэкапить, восстанавливается при работе приложения                 | Не требуется |

## Рекомендации

- **Периодичность:** не реже суточных бэкапов для `data/json` и медиа при активном редактировании контента.
- **Хранение:** копии держать вне сервера приложения (другой хост, объектное хранилище), с ограничением доступа.
- **Проверка:** периодически проверять восстановление из бэкапа (тестовый прогон).
- **Версионирование:** при использовании Git для контента (см. [content-versioning.md](content-versioning.md)) бэкап репозитория может дополнять файловые копии.

Конкретные инструменты (rsync, restic, облачные снимки и т.д.) выбираются под хостинг и инфраструктуру.
